Foundations — Complexity, Math Basics & Implementation Speed
Before any algorithm matters, you need automatic complexity analysis, fast I/O, and bug-free implementation under time pressure. This phase builds the reflexes — most people lose contest points to silly bugs, not missing knowledge.
⏱ 4–5 weeks Big-O · Arrays · Two Pointers · Sliding Window · Binary Search Foundation
// Complexity Cheat Sheet
Time Limit Heuristic
~10⁸ ops/sec. n≤10⁸→O(n), n≤10⁵→O(n log n), n≤500→O(n³)
Master Theorem
T(n)=aT(n/b)+f(n) → compare f(n) vs n^(log_b a)
Amortized Analysis
Total cost / n operations — used for dynamic arrays, union-find
Binary Search Invariant
lo, hi maintain: answer ∈ [lo, hi]; loop while lo < hi
Two Pointer Invariant
Monotonic window property must hold for both pointers to be valid
Space-Time Tradeoff
Precompute / memoize when same subproblem recurs ≥ O(log n) times

Core Data Structures
Stacks, queues, heaps, hash maps, trees, and the workhorse range-query structures: segment trees, Fenwick trees, and sparse tables. These are the same structures behind database indices, scheduler queues, and feature stores in production ML systems.
⏱ 6–7 weeks Heap · Segment Tree · Fenwick · DSU · Trie Intermediate
// Complexity Reference
Segment Tree
Build O(n), query/update O(log n), lazy prop O(log n)
Fenwick Tree (BIT)
Update O(log n), prefix query O(log n), space O(n)
Union-Find (DSU)
find/union O(α(n)) amortized with path compression + union by rank
Sparse Table
O(n log n) build, O(1) idempotent range query (min/max/gcd)
Heap Operations
insert/extract-min O(log n), peek O(1), build-heap O(n)
Trie
Insert/search O(L) per string of length L; space O(ALPHABET·N·L)

03
Graph Theory — Traversal, Shortest Paths & Flows
The single highest-leverage topic for an ML engineer. Graph traversal underlies dependency graphs in autodiff, GNN message passing IS graph traversal, and max-flow shows up in optimal transport and assignment problems used in modern ML pipelines.
⏱ 8–10 weeks BFS/DFS · Dijkstra · MST · Max Flow · SCC · LCA Core
// Graph Algorithm Complexity Reference
BFS / DFS
O(V + E) — shortest path on unweighted graphs (BFS), connectivity, cycles
Dijkstra (heap)
O((V + E) log V) — single-source shortest path, non-negative weights only
Bellman-Ford
O(VE) — handles negative weights, detects negative cycles
Floyd-Warshall
O(V³) — all-pairs shortest path, dense graphs, small V
Kruskal / Prim (MST)
Kruskal O(E log E), Prim O(E log V) with heap — minimum spanning tree
Max Flow (Dinic's)
O(V² E) general, O(E√V) on unit-capacity graphs (bipartite matching)

04
Dynamic Programming — The Career-Defining Skill
If there is one CP skill that transfers most directly to ML engineering, it's this. Bellman equations in RL are DP. Viterbi decoding is DP. Beam search is bounded DP. CYK parsing is DP. Mastering state design here makes you faster at decomposing every optimization problem you'll ever face.
⏱ 8–10 weeks Knapsack · LIS · Bitmask DP · Tree DP · Digit DP Core
// DP Pattern Reference
0/1 Knapsack
dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]]+val[i])
LIS (O(n log n))
Binary search on tails array; patience sorting technique
Bitmask DP (TSP)
dp[mask][i] = min cost to visit cities in mask, ending at i — O(2ⁿ·n²)
Tree DP
dp[node] computed from dp[children] via post-order DFS
Digit DP
dp[pos][tight][state] — count numbers with property up to N
Convex Hull Trick / Divide & Conquer Opt
Optimize O(n²) DP transitions to O(n log n) when cost function is convex/monotonic

06
Mathematics — Number Theory, Combinatorics & Game Theory
Modular arithmetic and combinatorics directly underpin cryptographic primitives, probability calculations in Bayesian ML, and the modular exponentiation used in efficient hashing. Game theory connects to multi-agent RL's Nash equilibrium concepts.
⏱ 6–7 weeks Modular Arithmetic · Combinatorics · Game Theory · FFT Advanced
// Number Theory Core Equations
Fermat's Little Theorem
a^(p-1) ≡ 1 (mod p) for prime p, gcd(a,p)=1 — modular inverse via a^(p-2)
Extended Euclidean
ax + by = gcd(a,b) — finds Bézout coefficients, modular inverse for any modulus
Sieve of Eratosthenes
O(n log log n) — primes up to n; linear sieve variant O(n)
CRT (Chinese Remainder Theorem)
Combine x ≡ aᵢ (mod mᵢ) for pairwise coprime mᵢ into unique solution mod ∏mᵢ
Nim / Sprague-Grundy
XOR of pile sizes ≠ 0 ⟹ first player wins (Nim); Grundy(state) = mex(reachable)
FFT Convolution
O(n log n) polynomial multiplication via Fast Fourier Transform / NTT
